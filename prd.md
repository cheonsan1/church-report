# 사역보고서 PRD (Product Requirements Document)

## 1. 프로젝트 개요
* **웹앱 이름:** 사역보고서
* **한 줄 설명:** 주일학교 및 교구 보고서를 모바일로 간편하게 제출하고, Supabase와 구글 스프레드시트에 자동 저장하여 대시보드에서 통합 조회할 수 있는 사역 관리 웹앱
* **앱 유형:** 내부 데이터 수집, 관리 및 대시보드 웹앱
* **예상 난이도:** 중 (관계형 데이터베이스 설계 및 외부 API 연동 필요)
* **핵심 목적:** 매주 발생하는 부서별/교구별 사역 보고서를 디지털화하여 데이터 취합의 비효율을 없애고, 담임목사가 전체 현황을 직관적으로 파악할 수 있도록 돕는다.

## 2. 사용자 시나리오
* **누가 사용하는가:** 주일학교 서기, 교구 목사 (작성자) / 담임목사 (열람자)
* **언제 사용하는가:** 주로 주일(일요일) 예배 직후
* **왜 사용하는가:** 종이나 개별 메시지로 흩어지는 보고를 일원화하고 체계적으로 관리하기 위해
* **사용자가 얻는 유익은 무엇인가:** 작성자는 스마트폰으로 빠르게 보고를 마칠 수 있으며, 열람자는 누락 없이 한눈에 전체 통계와 심방 스케줄을 파악할 수 있다.

## 3. 핵심 기능
* **반드시 필요한 기능:** 주일학교/교구별 맞춤 보고서 작성 폼, Supabase 및 구글 스프레드시트 자동 연동, 부서/교구 및 교구 성도 명단(구역 포함) 관리, 주간 심방 일정 통합 달력, 공통 비밀번호 기반의 대시보드 접근 제어
* **있으면 좋은 기능:** 교구 성도 명단 일괄 등록(엑셀 업로드) 기능
* **나중에 확장할 기능:** 작성자별 개별 로그인 도입, 전년도 대비 통계 분석 그래프

## 4. 기술 스택
* **프론트엔드:** Next.js (React), Tailwind CSS (단정하고 신뢰감 있는 UI 구현에 적합)
* **백엔드/데이터베이스:** Supabase (관계형 데이터 저장 및 명단 관리)
* **외부 연동:** Google Sheets API (데이터 백업 및 엑셀 뷰 제공용)
* **인증:** 대시보드 및 설정 화면용 단순 공용 비밀번호 (환경 변수 또는 Supabase Edge Functions 활용)
* **배포:** Vercel (GitHub 연동을 통한 자동 배포)

## 5. 화면 구성
* **보고서 작성 홈:** 주일학교 폼 / 교구 폼 선택 화면
* **주일학교 작성 화면:** 기본 정보, 예배 현황, 출석 통계, 사역 보고 입력 (모바일 최적화)
* **교구 작성 화면:** 구역별 명단 출결 체크, 유고사항 입력, 주중 심방 일정 입력 (모바일 최적화)
* **대시보드 화면:** 전체 출석 통계, 부서별 요약, 주중 통합 심방 스케줄 표 (PC 최적화, 비밀번호 필요)
* **설정 화면:** 부서/교구 관리, 교구별 구역 및 성도 명단 CRUD(추가/수정/삭제) (PC 최적화, 비밀번호 필요)

## 6. 상세 기능 명세
* **주일학교 보고서 제출:**
  * 사용자 행동: 기본/예배/출석/사역 데이터를 텍스트 및 숫자로 입력 후 제출
  * 입력값: 부서, 주차, 일시, 작성자, 예배담당자(사회/기도/찬양/설교자 등), 출석인원(교사/학생/새가족), 금주/차주 계획 및 기도제목
  * 결과 화면: "제출이 완료되었습니다" 메시지와 함께 홈으로 이동
* **교구 보고서 제출:**
  * 사용자 행동: 1~3교구 및 하위 구역 선택 시 미리 등록된 명단이 엑셀 스타일로 노출. 개별 출석/결석 토글 및 결석 사유(유고) 작성. 주중 스케줄 입력.
  * 저장되는 데이터: 개별 성도 출결 이력, 유고사항 텍스트, 심방 일정(일~토)
* **대시보드 브리핑:**
  * 사용자 행동: 공용 비밀번호 입력 후 접근하여 전체 현황 열람
  * 결과 화면: 각 부서 제출 상태, 총 출석 인원 합계, 교구별 주중 심방 스케줄(월~일 형태의 표) 출력

## 7. 디자인 가이드
* **전체 분위기:** 단정하고 신뢰감 있는 행정/관리 앱 스타일
* **색상 방향:** 흰색 배경(White) 바탕에 파란색(Blue) 포인트
* **글자 크기:** 가독성을 위해 본문 16px 이상, 입력 폼 간격 여유 확보
* **플랫폼 최우선:** * 폼 작성 화면: 모바일 최우선 (Mobile-First)
  * 대시보드 및 설정 화면: PC 최우선 (Desktop-First)

## 8. 제약 사항
* **보안:** 제출은 누구나 가능하되, 열람 및 설정은 1개의 공용 비밀번호로 통제
* **데이터 관리:** 모든 데이터는 Supabase에 원본 저장되며, 동시에 구글 스프레드시트에 자동 기입(Append) 되어야 함
* **추가 확인이 필요한 부분 (가정 사항):** * 구글 스프레드시트에 기입될 때, 주일학교 시트와 교구 시트를 분리하여 저장하는 것으로 가정함.
  * 공용 비밀번호는 코드 레벨(.env) 혹은 DB 설정 값으로 하드코딩하여 사용.

## 10. 화면별 와이어프레임 설명

1. **메인 화면 (Home)**
   * **목적:** 보고서 유형 선택
   * **주요 구성 요소:** [주일학교 보고서 작성] 버튼, [교구 보고서 작성] 버튼, [관리자/대시보드 접속] 하단 텍스트 링크
   * **사용자 행동:** 본인 역할에 맞는 버튼 탭

2. **주일학교 보고서 작성 화면**
   * **목적:** 주일학교 서기들의 데이터 입력
   * **주요 구성 요소:** 날짜 선택, 셀렉트박스(부서), 텍스트 인풋(담당자들), 숫자 인풋(출석 통계), 넓은 텍스트 에어리어(사역 보고). 하단 [제출하기] 버튼.
   * **사용자 행동:** 항목 순차 입력 후 제출

3. **교구 보고서 작성 화면**
   * **목적:** 교구 출결 및 심방 일정 입력
   * **주요 구성 요소:** * 상단: 교구 및 구역 드롭다운 선택
     * 중단: 구역 성도 명단 리스트 (이름 | 출결 토글 | 유고사항 입력 칸)
     * 하단: '주중 심방 일정 추가' 버튼
   * **사용자 행동:** 구역 선택 -> 명단 출결 체크 -> 심방 일정 추가 -> 제출

4. **대시보드 화면**
   * **목적:** 담임목사 브리핑 보드
   * **주요 구성 요소:** 주차 선택 필터, 총 출석 요약 카드, '이번 주 교구 심방 스케줄' 통합 테이블(일~토), 부서별 보고 텍스트 뷰.

5. **설정 화면**
   * **목적:** 기초 데이터 관리
   * **주요 구성 요소:** 부서/교구 관리, 교구/구역 카테고리별 성도 추가/삭제 리스트.

## 11. MVP 우선순위

* **1단계 (필수 기능):** DB 스키마 설계, 주일학교/교구 작성 폼 구현, 명단 기초 등록(단일 건), Supabase 및 구글 스프레드시트 저장 기능
* **2단계 (관리 및 보안):** 대시보드/설정 화면 공용 비밀번호 잠금, 대시보드 통계 및 심방 스케줄 시각화
* **3단계 (확장):** 엑셀 파일 명단 일괄 업로드, 개별 계정 로그인, 데이터 분석 차트

## 12. 개발자 (또는 AI 코딩 도구) 전달용 프롬프트

```text
Next.js (App Router), Tailwind CSS, Supabase, Google Sheets API를 사용하여 교회 사역보고서 웹앱을 개발해 줘. 배포는 Vercel을 사용할 거야. 모바일 작성 편의성과 PC 대시보드 가독성을 모두 고려한 반응형(Mobile-first Form, Desktop-first Dashboard)으로 만들어 줘. 디자인 테마는 흰색 배경에 신뢰감을 주는 파란색(Blue) 포인트 컬러를 적용해 줘.

[데이터베이스 스키마 요구사항 (Supabase)]
1. `departments` 테이블: id, name, type (sunday_school / parish)
2. `members` 테이블: id, department_id(교구), area(구역, 예: 1-1), name
3. `sunday_school_reports`: 부서 정보, 일시, 작성자, 예배담당자, 출석통계(교사/학생/새가족), 주간사역보고
4. `parish_reports`: 교구 정보, 일시, 작성자
5. `attendance_records`: report_id, member_id, status (출결), notes (유고사항)
6. `visitation_schedules`: report_id, day_of_week(일~토), target, description

[주요 페이지 요구사항]
- `/` (Home): '주일학교 보고서'와 '교구 보고서' 중 선택
- `/report/sunday-school`: 주일학교 폼
- `/report/parish`: 교구 폼. 교구/구역 선택 시 `members` 테이블에서 해당 명단을 불러와 동적 렌더링. 출석/결석 상태와 유고사항 입력. 하단에 주중 심방 일정을 요일별로 추가.
- `/dashboard`: 공용 비밀번호(`NEXT_PUBLIC_ADMIN_PASSWORD`) 잠금 적용. 성공 시 요약 통계와 '주중 심방 통합 달력(일~토)' 출력.
- `/admin`: 부서 및 성도 명단(members) CRUD 관리자 화면 (비밀번호 필요).
- [API Route]: 폼 제출 시 Supabase Insert 완료 후, Google Sheets API를 호출해 스프레드시트에 행(Row) 추가.

우선 데이터베이스 스키마 SQL 문을 작성해주고, 이어서 프로젝트 폴더 구조와 가장 핵심이 되는 교구 보고서 작성 페이지(`/report/parish`)의 React 컴포넌트 코드를 작성해 줘.
```
